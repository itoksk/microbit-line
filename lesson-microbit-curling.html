MICRO:BIT CURLING

Strategic Curling with micro:bit

Plan • Code • Curl

カーリングのように戦略を立て、micro:bit を走らせて得点を競うワークショップの初級編ガイドです。
MakeCode のブロックで信号機のシーケンス制御（正しい順番と時間）を体験しながら、「どのような組み合わせが必要か」「組み合わせをどう並べるか」を考えてコースを設計します。

01

ワークショップの全体像

INTRO

戦略と思考のウォームアップ

カーリングの試合づくりをヒントに、まずは「目標地点」「投げる順番」「重ねたい動き」を紙に書き出します。クラス全体で共有して、成功イメージをつくりましょう。

FLOW

設計 → プログラム → 調整

1. ルール理解 / 2. コース設計 / 3. サンプルの改造 / 4. 試走と軌道修正 / 5. 得点対決 の順に進めます。各ステップで役割を交代するとチーム全員が経験できます。

TEAM PLAY

役割リレーで集中力アップ

ストラテジスト（戦略係）・プログラマ・テストマネージャーを回しながら、考える力と実装力の両方を磨きます。困ったらペア同士でヒントを出し合いましょう。

ポイント
戦略は 1 つだけでなく複数案を用意しよう。「速く行く」「確実に止まる」「相手のストーンを押し出す」など、狙いを分解して考えると組み合わせが見えてきます。
02

シーケンス制御を体験しよう

信号機のように、あらかじめ決めた順番で動く仕組みを シーケンス制御 と呼びます。まずはライトの点灯順を作り替えながら、「順番と時間」をコントロールする感覚をつかみましょう。

SEQUENCE

決めた順番で安全を守る

  • 交通信号：青 → 黄 → 赤 の順を守ることで安全に横断できる。
  • 自動ドア：人を感知 → ドアを開ける → 通過を確認 → ドアを閉める。
  • 洗濯機：洗う → すすぐ → 脱水の順番を自動で繰り返す。

どれも「順番が崩れると危険・不便になる」ので、シーケンス制御で確実に動かしています。

TRY

信号シーケンスを読み解く

資料を見ながらライトの色と秒数を書き出し、ホワイトボードにフローチャートを作成しましょう。班のメンバー同士で順番の意味を説明し合うと理解が深まります。

チャートができたら、micro:bit で同じ順番を再現。青を長く、黄を短くするなど、順番と時間を変えると動きがどう変わるか観察します。

チャレンジ：MakeCode（日本語） を開いて、青5秒 → 黄2秒 → 赤4秒 → 歩行者点滅（3回）のシーケンスをブロックで組み立てよう。ブロックを配置する前に、ノートに「表示するアイコン」と「待つ時間」を並べてから作業するとミスが減ります。
  1. 新しいプロジェクトを作成し、最初に表示ブロックと「一時停止 (ms)」ブロックを交互に並べて青→黄→赤の順番を作る。
  2. 歩行者信号は「繰り返し」ブロックを 3 回に設定し、アイコン表示とクリア、短い待ち時間をはさみながら点滅させる。
  3. 完成したら実機またはシミュレーターで動作を確認し、秒数を微調整してみよう。
ヒントを見る（ブロック構成）
  • アイコンは「基本 > アイコンを表示」で選べます。色の代わりに形で区別しましょう。
  • 時間は秒ではなくミリ秒なので、5秒なら 5000、0.4 秒なら 400 を入力します。
  • 歩行者点滅は「基本 > LED をすべて消す」を組み合わせると、点灯→消灯のリズムが作れます。
コツ
順番と時間のセットを書き出してからプログラムを書くと、カーリングの走行プランにも応用しやすくなります。次のセクションではこの「順番づくり」を動きに置き換えていきます。
03

micro:bit カーリングの準備

安全に、確実に準備することが戦略実行の第一歩。写真を見ながら順番に行いましょう。

破損を防ぐには？

ケーブルを引っ張らない・ボードを持つときは両端を支える・持ち運ぶときは専用ケースへ。機材を大切に扱うことで、練習時間を最大化できます。
04

コートを理解し得点戦略を立てよう

ショットを成功させるには、コートのどこに止めたいかをイメージすることが大切です。

得点計算の流れ

チャレンジ：2 チームのストーン位置を紙に描き、「何点差？」を説明してみよう。説明には「最も近いストーン」「次に近いストーン」が必ず登場するように意識してみてください。
解答例をみる
  • ハウス中心に一番近いストーンをまず確認する（この例では青）。
  • 相手より内側にある自チームのストーンを数えると 2 個なので 2 点。
  • 次のエンドに向けて、赤チームはセンターガードを置くなど守りのシーケンスを考える。
05

戦略の組み合わせを考えよう

ゴール地点に届くまでの「並べ方」を考えるのがカーリングの醍醐味。コースを分割し、段階的に最適化します。

STRATEGY MAP

3 つの視点でプランを作成

  • スタート直後：まっすぐ加速する？ すぐ曲げる？
  • 中盤：相手ストーンを避けるガードコースは？
  • 終盤：どの角度で減速してハウスに止める？
TEAM TALK

役割ごとの声かけ例

ストラテジスト：「次の 20cm で右へ、最後は 40cm でブレーキ」
プログラマ：「速度 60→40→20 の 3 段階で再現してみよう」
テスト担当：「ラインから外れたら、どの区間を調整する？」

チャレンジ：ハウス中央に置かれた相手ストーンを押し出し、自分のストーンを残す戦略を考えてみよう。加速・横移動・減速の順序を言葉で説明してから、擬似コードを書いてみてください。
擬似コード例をみる
  • スタート：スピードを上げてストーンに勢いをつける。
  • 中盤：右タイヤを遅くするなどして相手ストーンの外側を回り込む。
  • 終盤：左タイヤを遅くして中心へ戻し、最後はブレーキ操作で押し出して停止。
06

ベースプログラムを理解しよう

MakeCode のサンプルプロジェクトを読み解き、戦略に合わせてブロックの値を調整します。

MAKECODE

サンプルプロジェクトからスタート

カーリング用 MakeCode プロジェクト を開き、「エディターで編集」をクリックするとブロックが読み込まれます。まずはそのまま実行し、どの順番でモーター速度が変化しているかを確認しましょう。

  • OPEN / MID / FINAL と名付けられたブロックの値を変更すると、直進・カーブ・減速のシーケンスが変化します。
  • 「モーターの速度を設定」ブロックで左右の値を変えると、カーブのきつさを調整できます。
  • 順番ごとに「一時停止 (ms)」ブロックを入れておくと、走行ログと比較しやすくなります。
チャレンジ：サンプルの「OPEN」の速度を 50 に下げ、「MID」の待ち時間を 520 ミリ秒に伸ばすとコースはどう変わるか？ チームで予測を書き出してから試走しよう。
予測のヒントを見る
  • 初速を抑えると、直進区間が短くなりガード付近で早めに曲がり始める。
  • MID の時間を長くすると、右カーブ区間が広がり相手ストーンの外側を回り込みやすい。
  • そのぶん FINAL での戻しが不足する可能性があるので、左右の速度差や待ち時間で微調整しよう。
07

試走と軌道修正のコツ

テスト走行で得たデータを基に、数値を微調整します。スプレッドシートやホワイトボードにログを残すと振り返りが楽になります。

チャレンジ：テスト走行 3 回分のログを取り、「どの区間で右へ流れたか」「修正案は何か」を書き出してみよう。修正後のコードも記録に残すと次のチームがスムーズに引き継げます。
ログ記入例をみる
  • Run1：MID の後半で右へずれた → 「MID の左モーター速度」を +8。
  • Run2：FINAL で止まりきらない → 待ち時間を +80、左右の速度を -5。
  • Run3：ハウス内で停止成功 → 次は OPEN の速度を +20 にして押し出し戦略を試す。
08

ゲーム形式で仕上げよう

全チームが 3 エンド戦を行い、戦略 → 実行 → 振り返りのサイクルを体験します。審判係も交代でチャレンジしてみましょう。

SCORING

審判係の進め方

① スロー後は必ずストーン位置を写真やメモに記録。
② 得点表を更新しながら、誰が次のエンドでハンマー（最後の投球権）かを伝えます。
③ フェアプレイの声かけも審判の大切な役割です。

REFLECT

ふりかえりの質問カード

・予定通りに動いたステップはどこ？
・調整が必要だったのはどの組み合わせ？
・次に挑戦したい戦略は？ など、短時間で共有できる問いを用意しておきましょう。

チャレンジ：ストーンまでの距離カードを並べ替えて得点を数えるボードを作ろう。並べ替えの手順をホワイトボードに書いたら、同じ考え方で MakeCode の変数やリストブロックを組み合わせて簡単なスコアカウンターに挑戦してみても OK！
並べ替え手順のヒント
  • 自チームと相手チームの石の距離カードを小さい順に並べる。
  • 中心に一番近いカードが相手になるまで、自チームのカードを 1 枚ずつ数える。
  • 数えた枚数が得点。MakeCode で自動化するなら、繰り返しブロックと比較ブロックを使って同じ順番を再現しよう。
次のステップ
初級編が終わったら、ラインセンサーや距離センサーを追加して自動オートメーションに挑戦！ ハイブリッド戦略で得点を伸ばしてみましょう。
